# CLAUDE.md — 辅食排敏助手（微信小程序）

## 项目概述

一款专注于婴儿辅食记录与排敏管理的微信小程序。核心差异点：图片化食物选择（老年人友好）、智能排敏计划引擎、过敏溯源分析。

目标用户：6-24月龄婴幼儿的照护者（年轻父母 + 祖辈 + 月嫂/育儿嫂）。

## 技术栈

- **前端框架**：微信小程序原生开发（WXML + WXSS + JS）
- **UI 组件库**：WeUI 或 Vant Weapp
- **后端**：微信云开发（云函数 + 云数据库 + 云存储）
- **语言**：JavaScript（ES6+）
- **包管理**：npm
- **状态管理**：小程序自带的 globalData + 页面间通信

### 为什么选微信云开发

- 免服务器运维，适合个人/小团队快速验证
- 天然支持微信登录，无需额外鉴权
- 云数据库是文档型（类 MongoDB），适合灵活的食物记录数据结构
- 支持实时数据库推送，方便家人协同

## 核心功能模块详细需求

### 模块一：首页 — 今日辅食看板 (pages/index)

**功能清单：**
- 显示今天的排敏任务："今天要吃 → 蛋黄（第2天/共3天）"
- 显示今天已记录的辅食列表（图片+名称+时间）
- 过敏预警：如果最近有"疑似过敏"记录，顶部显示橙色提示条
- 快速记录入口：大按钮"记录辅食"跳转到 record 页
- 底部 tabBar：首页 | 记录 | 计划 | 我的

**UI 设计要点：**
- 简洁清爽，大字体（考虑老年用户）
- 今日任务卡片要醒目，用柔和的彩色背景
- 已记录列表用食物图片展示，不用纯文字

### 模块二：辅食记录 — 图片化食物选择 (pages/record)

**功能清单：**
- 顶部分类标签栏：谷物 | 蔬菜 | 水果 | 肉蛋 | 乳制品 | 豆制品 | 海鲜 | 其他
- 每个分类下展示食物卡片网格（图片 + 名称），点击即选中
- 支持搜索框快速搜索食物名
- 选中食物后弹出记录面板：
  - 时间（默认当前，可修改）
  - 食用量（少量 / 正常 / 多）三个按钮选择
  - 反应（正常 / 疑似过敏 / 过敏）三个按钮选择
  - 备注（可选文字输入）
  - 拍照记录（可选，拍摄实际辅食照片）
- 系统自动判断是否为首次食用，首次食用时提示"这是宝宝第一次吃XX哦"
- 支持一次记录多种食物
- 保存后返回首页，自动刷新今日记录

**UI 设计要点：**
- 食物卡片要大（最少 80x80px），图片清晰
- 卡片网格每行 3-4 个，方便点击
- 选中状态要有明显的视觉反馈（边框变色、打勾）
- 记录面板用底部弹出半屏的方式，不跳转新页面

### 模块三：排敏计划 — 日历视图 + 计划管理 (pages/plan)

**功能清单：**
- 日历视图，每天的格子上显示当天计划食物的小图标
- 颜色编码：绿色=已通过、黄色=测试中、灰色=待开始、红色=过敏
- 点击某天可查看当天详细计划和记录
- 排敏队列管理：
  - 查看待测试食物列表（可拖拽排序）
  - 添加新食物到排敏队列
  - 设置每种食物的测试天数（默认3天，可修改）
  - 设置每天最多引入几种新食物（默认1种）
- 已通过食物清单（绿色列表）
- 过敏食物清单（红色列表，标注复测日期）
- 自动推算：根据当前进度 + 排敏规则，计算出未来每天该吃什么

**核心算法 (utils/plan-engine.js)：**
```
输入：排敏队列、测试天数规则、当前日期
输出：未来N天的每日计划

逻辑：
1. 取队列中 status="测试中" 的食物，检查是否已完成全部测试天数
2. 如果已完成 → 标记为"已通过"，取队列下一个食物开始测试
3. 如果当前没有测试中的食物 → 取队列第一个"待开始"的食物，设置开始日期为明天
4. 生成未来7-14天的计划视图
5. 每天的计划 = 正在测试的新食物 + 已通过的安全食物（可选搭配）
```

### 模块四：过敏溯源 (pages/trace)

**功能清单：**
- 入口："宝宝过敏了"按钮
- 记录过敏症状：
  - 症状类型（多选）：皮疹、腹泻、呕吐、湿疹加重、鼻塞、其他
  - 严重程度：轻度 / 中度 / 重度
  - 发生时间
  - 备注
- 系统自动分析：
  - 回溯过去72小时内所有"首次食用"的食物
  - 按时间远近和过敏风险排列"可疑食物"列表
  - 高亮显示最可疑的食物
- 用户确认：
  - 选择确认哪种食物过敏
  - 确认后：该食物加入黑名单，排敏计划自动跳过
  - 可设置复测提醒（默认3个月后）
- 过敏历史时间线：
  - 按时间倒序展示所有过敏事件
  - 每条记录显示：日期、症状、确认的过敏食物、是否已复测

**核心算法 (utils/allergy-trace.js)：**
```
输入：过敏发生时间、观察窗口（默认72小时）
输出：可疑食物排名列表

逻辑：
1. 查询过敏发生前72小时内的所有辅食记录
2. 筛选出 isFirstTime=true 的记录（新食物优先可疑）
3. 按以下权重排序：
   - 首次食用 > 非首次食用
   - 食物本身过敏风险（高 > 中 > 低）
   - 时间越近 > 时间越远
4. 返回带置信度标签的可疑食物列表
```

### 模块五：设置 (pages/settings)

**功能清单：**
- 宝宝信息管理（姓名、生日、头像）
- 排敏规则设置：
  - 每种新食物连续食用天数（默认3天）
  - 观察窗口时长（默认72小时）
  - 每天最多引入新食物种类数（默认1种）
- 家人协同管理：
  - 生成邀请二维码/链接
  - 查看已加入的家人列表
  - 移除家人
- 数据导出（生成 PDF 报告，去医院体检用）
- 关于/帮助

**谷物类（6月+）：** 大米/米粉、小米、燕麦、玉米、面条、馒头、面包
**蔬菜类（6月+）：** 胡萝卜、南瓜、土豆、红薯/紫薯、西蓝花、菠菜、白菜、西葫芦、冬瓜、山药、莲藕、番茄、豌豆

## 编码规范

- 使用 ES6+ 语法
- 组件命名使用 kebab-case（如 `food-picker`）
- 页面和组件的 JS 文件使用驼峰命名函数
- CSS 类名使用 BEM 规范（如 `.food-card__image--selected`）
- 云函数统一返回格式：`{ code: 0, data: {}, message: "success" }`
- 错误处理：所有异步操作使用 try-catch，用户侧显示友好提示
- 日期统一使用 ISO 8601 格式存储，展示时转为本地时间
- 注释语言：中文

## 注意事项

- 图片资源：MVP 阶段食物图片可使用开源素材或 AI 生成的简洁插画风格图标，保持风格统一
- 性能：食物图库数据放本地（data/food-library.js），避免每次联网加载
- 离线体验：核心记录功能应支持离线暂存，联网后自动同步
- 字体大小：全局最小字体 14px，重要信息 16-18px（照顾老年用户）
- 颜色对比度：确保所有文字与背景的对比度满足 WCAG AA 标准
- 小程序包体积：注意图片压缩，单个食物图标控制在 20KB 以内
